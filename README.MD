#### Алгоритм жадного распределения ресурсов

##### Этап 1: Предварительная обработка данных

  Шаг 1.1: Инициализация рабочих переменных

    selected_projects = []    # Результирующий список выбранных проектов
    used_resource = 0         # Суммарное количество использованных ресурсов
    total_profit = 0          # Общая прибыль от выбранных проектов

  Шаг 1.2: Расчет показателей эффективности

  Для каждого проекта project в списке projects вычисляется:
    эффективность = project['profit'] / project['resource']

  Этот показатель определяет, сколько единиц прибыли приносит каждая единица вложенного ресурса.

  Шаг 1.3: Сортировка проектов

  Проекты сортируются по убыванию эффективности с использованием функции:

    projects_sorted = sorted(projects, 
                           key=lambda x: x['profit'] / x['resource'], 
                           reverse=True)

  Параметр reverse=True обеспечивает порядок от наибольшего к наименьшему.

##### Этап 2: Процесс отбора проектов

  Шаг 2.1: Последовательный обход отсортированного списка

  Алгоритм проходит через каждый проект в списке projects_sorted в порядке убывания эффективности.

  Шаг 2.2: Проверка условия включения проекта

  Для каждого рассматриваемого проекта выполняется проверка:
    if used_resource + project['resource'] <= total_resource:

  Логика проверки:

  · used_resource - текущее количество использованных ресурсов
  · project['resource'] - ресурсы, требуемые для текущего проекта
  · total_resource - общий доступный ресурс

  Если сумма не превышает общий лимит, проект может быть включен.

Шаг 2.3: Включение проекта в решение

Если условие выполняется:

1. Проект добавляется в список selected_projects
2. Значение used_resource увеличивается на project['resource']
3. Значение total_profit увеличивается на project['profit']

Шаг 2.4: Пропуск проекта

Если условие не выполняется (не хватает ресурсов), проект пропускается, и алгоритм переходит к следующему проекту в отсортированном списке.

Этап 3: Формирование результатов

Шаг 3.1: Подготовка выходных данных

Алгоритм возвращает кортеж из трех элементов:
return selected_projects, total_profit, used_resource

Шаг 3.2: Интерпретация результатов

· selected_projects - список проектов, выбранных для реализации
· total_profit - суммарная прибыль от всех выбранных проектов
· used_resource - общее количество использованных ресурсов
