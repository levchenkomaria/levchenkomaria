#### Алгоритм жадного распределения ресурсов

Алгоритм решает задачу оптимального распределения ограниченного объёма ресурсов между проектами, чтобы максимизировать общую прибыль. Каждый проект требует определённое количество ресурсов и приносит прибыль.

###### 1. Ввод данных

Сначала задаётся список проектов — например, 
```python
  projects = [

    {resource: 10, profit: 50}, 

    {resource: 15, profit: 80}, 

    {resource: 20, profit: 90}, 

    {resource: 8, profit: 40}

  ].
```
Также задаётся общий доступный ресурс, например total_resource = 40.

###### 2. Расчёт эффективности проектов

Для каждого проекта вычисляется прибыль на единицу ресурса, то есть отношение прибыли к количеству требуемых ресурсов. Это реализуется через выражение 

```python
p[“ratio”] = p[“profit”] / p[“resource”].
```
Таким образом, мы определяем, какой проект наиболее выгоден в пересчёте на один ресурс.

###### 3. Сортировка по убыванию прибыльности

После этого проекты сортируются по рассчитанному показателю эффективности с помощью 

```python
sort(key=lambda x: x[“ratio”], reverse=True)
```
В результате первым будет рассматриваться проект, приносящий наибольшую прибыль на единицу ресурса.

###### 4. Выбор проектов
Алгоритм последовательно проходит по отсортированному списку. Если текущий проект можно полностью профинансировать, выполняется добавление ресурсов и прибыли: 

```python
used_resource += p[“resource”], total_profit += p[“profit”]
```
Если оставшихся ресурсов недостаточно, проект берётся частично — в этом случае вычисляется доля part = remain / p[“resource”], и прибыль увеличивается на p[“profit”] * part. Таким образом, ресурсы используются максимально эффективно.

###### 5. Вывод результата

В конце программа выводит список выбранных проектов с указанием их ресурса и прибыли, а также итоговое количество использованных ресурсов и общую прибыль.

#### Пример ввода/вывода

Пример ввода:

```python
projects = [
    {"resource": 10, "profit": 50},
    {"resource": 15, "profit": 80},
    {"resource": 20, "profit": 90},
    {"resource": 8,  "profit": 40}
]

total_resource = 40
```
Пример вывода:

```python
Selected projects:
  - Resource = 15, Profit = 80.00
  - Resource = 10, Profit = 50.00
  - Resource = 8, Profit = 40.00
  - Resource = 7, Profit = 31.50

Total used resource: 40
Total profit: 201.50
```

#### Временная сложность и объяснение

O(n) + O(n \log n) + O(n) = O(n \log n)

Объяснение:

Алгоритм состоит из трёх основных шагов:

 • Вычисление эффективности проектов (profit/resource) для всех n проектов. Это один проход по списку, поэтому занимает O(n) времени.

 • Сортировка проектов по убыванию эффективности. Сортировка массива из n элементов в среднем и худшем случае требует O(n log n) операций. Это самый затратный шаг в алгоритме.

 • Выбор проектов до исчерпания ресурсов. Алгоритм проходит по уже отсортированному списку, добавляя проекты или их части, что снова занимает O(n) времени.

Поскольку линейные операции (вычисление эффективности и выбор проектов) выполняются быстрее, чем сортировка, доминирующий член сложности — сортировка. Поэтому общая временная сложность алгоритма равна O(n log n).
